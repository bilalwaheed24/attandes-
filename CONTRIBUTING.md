# Contributing to SAMS

Thank you for your interest in contributing to the Smart Attendance Management System!

## Guidelines

### Code Standards
- Use TypeScript for all new code
- Follow existing code style
- Add JSDoc comments for functions
- Write unit tests for new features

### Commit Messages
- Use clear, descriptive commit messages
- Follow conventional commit format
- Example: `feat: add attendance export to CSV`

### Pull Requests
1. Create a feature branch from `main`
2. Implement your changes
3. Add tests for new functionality
4. Update documentation
5. Submit PR with detailed description

## Development Setup

See [QUICKSTART.md](QUICKSTART.md) for setup instructions.

## Code of Conduct

- Be respectful and professional
- Provide constructive feedback
- Focus on the code, not the person
- Help maintain a positive community

## Questions?

Please refer to the [documentation](docs/) or create an issue.
